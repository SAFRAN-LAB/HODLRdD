Wrap a raw polymorphic pointer in a shared\+\_\+ptr to its true type

The wrapped pointer will not be responsible for ownership of the held pointer so it will not attempt to destroy it; instead the refcount of the wrapped pointer will be tied to a fake \textquotesingle{}ownership pointer\textquotesingle{} that will do nothing when it ultimately goes out of scope.

The main reason for doing this, other than not to destroy the true object with our wrapper pointer, is to avoid meddling with the internal reference count in a polymorphic type that inherits from std\+::enable\+\_\+shared\+\_\+from\+\_\+this.
